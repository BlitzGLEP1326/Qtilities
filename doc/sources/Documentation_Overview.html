%Qtilities Modules:
Logging Module: Provides logging capabilities to be used in any Qt application.
To add the correct paths to your include paths and link to the correct libraries, add this in your .pro %file: \code QTILITIES += logging; include(Qtilities_Path/src/Qtilities.pri) \endcode To include all files in this module, add this to your .h/.cpp files: \code #include // In your .cpp files, use this namespace: using namespace QtillitiesLoggingModule; \endcode

To add the correct paths to your include paths and link to the correct libraries, add this in your .pro %file: \code QTILITIES += core; include(Qtilities_Path/src/Qtilities.pri) \endcode To include all files in this module, add this to your .h/.cpp files: \code #include // In your .cpp files, use this namespace: using namespace QtillitiesCoreModule; \endcode
CoreGui Module: Provides the core %Qtilities GUI classes.
To add the correct paths to your include paths and link to the correct libraries, add this in your .pro %file: \code QTILITIES += coregui; include(Qtilities_Path/src/Qtilities.pri) \endcode To include all files in this module, add this to your .h/.cpp files: \code #include // In your .cpp files, use this namespace: using namespace QtillitiesCoreGuiModule; \endcode
Project Management Module: Provides project management classes.
To add the correct paths to your include paths and link to the correct libraries, add this in your .pro %file: \code QTILITIES += project_management; include(Qtilities_Path/src/Qtilities.pri) \endcode To include all files in this module, add this to your .h/.cpp files: \code #include // In your .cpp files, use this namespace: using namespace QtillitiesProjectManagementModule; \endcode

The %Qtilities set of libraries has lots of features and classes. This section provides insights into some of the things you can do with the libraries and shows you where to get more information about a topic.
Object & Context Management Features:
Easily build, serialize and view hierarchical tree structures: Using the Qtilities::Core::Observer class it is possible to easily create tree structures. Tree items can appear multiple times in the tree hierarchy. It is possible to do a binary export on any tree structure and to monitor and control the modification state of the tree. Trees can be compared with each other and verbose information about trees can be obtained and exported. Trees can be viewed instantly in either a tree view or a table view in which the user can push up and down in the tree structure.
Further reading:
- \ref page_tree_structures

- \ref page_observers
\sa Qtilities::Core::Observer, Qtilities::Core::Interfaces::IModificationNotifier, Qtilities::Core::Interfaces::IExportable, Qtilities::Core::ObserverRelationalTable, Qtilities::CoreGui::ObserverWidget
Manage object lifetimes: The subject-observer pattern implementation in %Qtilities (the Qtilities::Core::Observer class) provides numerous ways to manage the lifetime of objects. A global object pool is available in which objects can be registered and inspected for interface implementations etc. These functions are all available through a object manager singleton.
Further reading:
- \ref page_object_management

- \ref page_observers
\sa Qtilities::Core::Observer, Qtilities::Core::QtilitiesCore, Qtilities::Core::ObjectManager
Group objects into different customizable contexts and easily view the contents of a context: The Observer class provides a flexible way to group objects into different contexts and manage these contexts accordingly. Contexts can be completely customized (attaching and detaching policies etc) and easily viewed in either tree views and/or table views. Observer properties allows properties to be assigned to objects on a per context basis or properties can be shared between different contexts.
Further reading:
- \ref page_observers

- \ref page_observer_widgets
\sa Qtilities::Core::Observer, Qtilities::Core::QtilitiesCore, Qtilities::Core::AbstractSubjectFilter, Qtilities::Core::ObserverProperty, Qtilities::Core::SharedObserverProperty, Qtilities::CoreGui::ObserverWidget, Qtilities::CoreGui::ObjectScopeWidget, Qtilities::CoreGui::ObjectPropertyBrowser
Create and manage factories: Using the factory related classes it is easy to create multiple object factories in your code. Factory base classes have the single requirement that they should inherit from QObject. Factories can be registered with the object manager.
Further reading:
\ref page_factories

\sa Qtilities::Core::Factory, Qtilities::Core::Interfaces::IFactoryData, Qtilities::Core::ObjectManager
Application Level Functionality:
Manage shortcuts and actions in your applications: The %Qtilities libraries provides classes to manage actions and shortcuts in your applications. Actions and shortcuts can be linked to multiple contexts and respond according to the active context.
